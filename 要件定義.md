## 0. このドキュメントの位置づけ

- アプリの**コンセプト → 機能 → 画面 → データ → 開発フェーズ**を一気通貫で定義する。
- まずは **Webプロトタイプ**（クリック可能なデモ）の仕様を固め、その後 **iOS実機アプリ**へ展開する。
- デザインは同フォルダ内 `yumekana-portal/index.html` のトンマナを全面的に踏襲する。

## 1. コンセプト / 目的

「スマホを開くたびに英語で1問。答えるまで先に進めない」ことで、**英語アウトプットを強制的に習慣化**する学習アプリ。

- ねらい: 「勉強しよう」という意思に頼らず、**生活動線に英語を割り込ませる**ことで継続率を最大化する。
- 学習サイクル: **出題（割り込み）→ 英作文で回答 → AIが添削 → 後でまとめて復習**。
- ターゲット像（★要確認）: 日本語話者の英語学習者（初級〜中級／CEFR A1〜B1 を主想定）。

## 3. ⚠️ 最重要の技術制約（iOSの「ロック」について）

ご要望の「**回答するまで携帯が使えない**」は、iOS（iPhone）では**OS仕様により第三者アプリが他アプリの上に強制オーバーレイを出すこと・端末をロックすることができません**。実現できるのは次のいずれかです。

| やりたいこと | iOSでの実現手段 | 実現度 |
|---|---|---|
| 定期的に学習を促す | **ローカル通知**（スケジュール送信） | ○ ただし無視可能・強制不可 |
| 「使うまで答えさせる」 | **Screen Time / Family Controls（ManagedSettings＋DeviceActivity）** で対象アプリの起動を**シールド（ブロック）**し、シールドのボタンから本アプリへ誘導→回答→解除 | △ "アプリ起動ゲート"型。常時オーバーレイではない |
| 5分ごとの強制割り込み | （バックグラウンド常駐は不可） | ✕ 通知での代替のみ |

### 本アプリでの「ロック」の再定義（iOS版）
> **「強制ロック」＝「指定アプリを開こうとするとシールドが出て、英語1問に答えるまでそのアプリを開けない集中モード」** とする。
> - 例: SNS等を"対象アプリ"に設定 → ユーザーがそれを開く → シールド表示 → 本アプリで回答 → 一定時間アンロック。
> - 加えて**定期ローカル通知**で「今日の1問」をリマインド。

- `★要確認`: この"集中モード"の体験で当初の狙いを満たせるか（後述 §13-Q1）。
- Family Controls は **Apple の特別な entitlement 申請**が必要（個人開発者でも申請可、審査あり）。
- 補足: **Webプロトタイプ／PWA では通知もロックも不可**（アプリを開いている間だけ動作）。プロトタイプではこれを**アプリ内フルスクリーンのロック画面**で"疑似再現"する。

## 4. 機能要件（Functional Requirements）

優先度: ◎=MVP必須 / ○=重要 / △=後回し可

| ID | 機能 | 概要 | 優先 |
|---|---|---|---|
| F1 | オンボーディング/プロフィール | 名前・レベル・興味カテゴリ・出題設定の初期登録 | ◎ |
| F2 | 出題（ロック画面） | 割り込みで英語の質問を1問表示。回答するまで閉じられない | ◎ |
| F3 | 回答入力 | 英文をテキスト入力して送信（将来: 音声入力） | ◎ |
| F4 | AI添削 | 回答を添削（修正文・指摘・語彙提案・スコア・総評） | ◎ |
| F5 | 添削結果表示 | 送信直後に結果を表示。原文と修正文の差分を見やすく | ◎ |
| F6 | 復習タブ（履歴） | 過去の質問・回答・添削を一覧／検索／フィルタ | ◎ |
| F7 | 単語・表現帳 | 添削で出た良い表現・単語を保存し復習（フラッシュカード） | ○ |
| F8 | ダッシュボード | 連続日数(streak)・回答数・平均スコア推移・レベル進捗 | ○ |
| F9 | 出題ロジック | バンク＋AI生成のハイブリッド／弱点復習出題 | ○ |
| F10 | 設定 | 出題間隔・通知ON/OFF・難易度・対象アプリ(iOS)・AI比率 | ○ |
| F11 | 通知/シールド(iOS実機) | ローカル通知＋Screen Timeシールド | ○(実機) |
| F12 | AI相談チャット | 学習相談（yumekanaのFAB流用、任意） | △ |

### F2/F3/F5 出題〜回答〜添削フロー（コア体験）
1. トリガー（時間間隔 or アプリ起動 / 実機はシールド経由）で**ロック画面**を表示。
2. 質問を1問提示（日本語の補助文付き・任意）。「スキップ」可否は `★要確認`（§13-Q2）。
3. ユーザーが英文を入力 → 送信。
4. AI添削を実行（ローディング表示）。
5. **添削結果**を表示: ①修正後の自然な英文 ②指摘リスト ③良い点 ④語彙/言い回し提案 ⑤スコア ⑥日本語総評。
6. 「保存して閉じる」でロック解除。結果は履歴に自動保存。

### F4 AI添削の出力仕様（Gemini）
入力: `質問文 / ユーザー回答 / ユーザーのレベル`。出力は構造化JSON（プロトのモックも同形式）:
```jsonc
{
  "corrected": "自然に直した英文",
  "score": 0,                 // 0-100（目安）
  "level_estimate": "A2",      // CEFR目安
  "praise": "良かった点（日本語）",
  "summary": "総評（日本語・2〜3文）",
  "issues": [
    {
      "original": "原文の該当部分",
      "fixed": "修正例",
      "type": "grammar|vocab|usage|spelling|naturalness",
      "explanation": "なぜ直すのか（日本語）"
    }
  ],
  "suggestions": [             // より良い言い回し
    { "expression": "英語表現", "note": "ニュアンス（日本語）" }
  ],
  "vocab": [                   // 覚えると良い単語/熟語
    { "word": "単語/熟語", "meaning": "意味（日本語）", "example": "例文" }
  ],
  "region_idiom": {            // 地域別イディオム（定期的に付与・無い回もある）
    "region": "US|CA|UK|AU", "expression": "イディオム",
    "meaning": "意味（日本語）", "example": "例文"
  }
}
```

## 7. 出題ロジック（ハイブリッド）

- **質問バンク**: カテゴリ（日常/食事/仕事/趣味/旅行/感情/ニュース…）× 難易度（A1〜B2）でタグ付けした固定質問群。安定・低コスト・オフライン可。
- **AI生成**: ユーザーのレベル・興味・**過去の弱点**（よく間違える文法/語彙）に合わせて Gemini が新規質問を生成。マンネリ防止。
- **選択ルール（案）**: 既定はバンクから出題。`aiRatio`（設定）に従い一定割合でAI生成。週1回など「弱点リカバリ出題」を混ぜる。同一質問の連続を避ける。
- `★要確認`: 初期バンクの規模・カテゴリ・例文の言語監修（誰が用意するか）。

## 8. 非機能要件

- **デザイン**: §10。yumekanaのグラスモーフィズム/ミント×ティールを継承。
- **パフォーマンス**: プロトは単一HTMLで即時表示。AI添削は3〜8秒想定 → ローディング/スケルトン表示。
- **オフライン**: バンク出題と履歴閲覧はオフライン可（AI添削はオンライン必須）。
- **データ保持**: プロトは端末内 `localStorage`。実機本番はクラウド同期＋バックアップ（将来）。
- **プライバシー**: 回答文＝個人的内容を含む。AI送信内容と保存方針を明示（同意導線）。`★要確認`。
- **アクセシビリティ**: 文字サイズ・コントラスト・スクリーンリーダー配慮。

## 9. AI / セキュリティ（Gemini API）

- **APIキーは絶対にクライアント（ブラウザ/アプリ）に直書きしない**。直書きするとブラウザの開発者ツール等から漏洩する。
- 推奨構成: **軽量バックエンド（サーバーレス関数：Cloudflare Workers / Vercel / Cloud Functions 等）をプロキシ**にして、キーはサーバ側の環境変数に保持。クライアントは自前APIを叩く。
- そのため**プロトタイプはまずモック添削**（サンプル数種＋簡易ルール）で体験を完成させ、`fetchFeedback()` を1関数だけ差し替えれば本物のGeminiに接続できる設計にする。
- キーは `.env` / シークレットマネージャで管理し、リポジトリにコミットしない（`.gitignore`）。
- モデル: Gemini系（例 `gemini-2.x-flash` 等、コスト/速度で選定）`★要確認`。
